Goal: Start the architecture with the durable ingestion and raw persistence path.
Do This Live:
1. Say: "I'll start with the write path: provider data enters through an ingest API, is durably buffered, then written to immutable raw readings."
Write: `External Provider -> Ingest API -> Kafka readings topic -> Raw Writer -> Postgres raw_readings`
Why: This covers burst buffering and no accepted reading is silently lost; raw readings remain the source of truth for corrections and reprocessing.
2. Say: "The API returns 202 only after Kafka confirms the write, and the raw writer commits offsets only after the database insert."
Write: `202 after Kafka ack; offset commit after raw insert`
Why: That gives at-least-once delivery with an idempotent raw sink, without needing exactly-once messaging.
